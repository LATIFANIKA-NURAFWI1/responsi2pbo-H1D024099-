Pertanyaan Refleksi
Setelah menyelesaikan kode, jawab pertanyaan berikut:

1. Apa itu abstract class dan Kenapa kita membutuhkan Abstarct class?
2. Apa perbedaan utama antara abstract class dan class biasa dalam pemrograman berorientasi objek? 
3. Mengapa subclass wajib mengimplementasikan semua abstract method dari parent class?
4. Mengapa abstract method tidak boleh memiliki isi implementasi di dalam abstract class?
5. Bagaimana abstract class dapat mendukung konsep polimorfisme, dan mengapa objek yang direferensikan melalui tipe abstract class tetap dapat menjalankan perilaku berbeda pada masing-masing subclass?

Jawaban:
1. Class yang tidak bisa dibuat objeknya secara langsung dan berfungsi sebagai blueprint bagi class lain. Dipakai saat butuh struktur dasar yang harus dilengkapi subclass.
2. Abstract class punya abstract method, tidak bisa diinstansiasi. Class biasa bisa dibuat objek dan methodnya lengkap
3. Karena abstract method tidak memiliki implementasi, maka subclass harus mengimplementasikannya.
4. Karena tujuannya memang hanya untuk mendefinisikan "apa yang harus dilakukan", bukan "bagaimana caranya". Implementasi detailnya diserahkan sepenuhnya kepada subclass
5. Referensi abstract bisa menunjuk ke berbagai subclass, dan masing-masing menjalankan perilaku uniknya
